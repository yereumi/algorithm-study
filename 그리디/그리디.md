# 그리디 (Greedy)

## 개념
- 매 단계에서 지금 당장 가장 좋아 보이는 선택을 하는 알고리즘
- 전체 최적해를 보장하는 문제에서만 사용 가능
- 선택을 되돌리지 않음
- 보통 **정렬 + 반복 선택** 구조

## 사용 상황
- 각 단계의 선택이 미래 선택에 영향을 주지 않을 때
- 부분 최적 선택 &rarr; 전체 최적이 성립할 때
- 핵심 키워드
  - 가장 큰 / 가장 작은
  - 최대한 많이
  - 최소 비용
  - 최소 횟수

### 그리디가 성립하는 조건
#### 1. 탐욕적 선택 속성 (Greedy Choice Property)
- 현재의 최선 선택이 전체 최적해에 포함됨

#### 2. 최적 부분 구조 (Optimal Substructure)
- 전체 문제의 최적해가
부분 문제의 최적해로 구성됨

&rarr; 두 조건이 모두 만족해야 그리디 사용 가능

## 구현 포인트
#### 1. 선택 기준을 정렬 기준으로
- 그리디의 핵심은 선택 기준
- 대부분 문제는 정렬 한 번으로 결정됨

#### 2. 현재 선택이 되돌릴 필요가 없는지 확인
- 선택 후에 다시 바꾸면 → 그리디 아님
- if / continue 구조로 즉시 확정

#### 3. 반례 생각
- 작은 입력으로 손으로 시뮬레이션
- 정렬 기준이 바뀌면 결과가 달라지는지 확인

#### 4. 지금 최선이 무엇인지 변수로 관리
- 마지막 선택 위치
- 현재 누적 비용
- 현재 종료 시각 등

## 시간복잡도
- 정렬 있는 경우: **O(N log N)**
- 정렬 없이 한 번만 순회: **O(N)**
- 우선순위 큐 사용하는 경우: **O(N log N)**

## 실수 포인트
- 그리디처럼 보인다고 무조건 적용
- 정렬 기준 잘못 설정 / 오타
- 비교 연산자 방향 실수 (<, <=)
- 누적 변수 초기화 안 함
- break/continue 위치 실수
- 반례 검증 안 함
