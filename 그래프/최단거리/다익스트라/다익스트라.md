# 다익스트라 (Dijkstra)

## 개념
- 하나의 시작 정점에서 다른 모든 정점까지의 최단 거리를 구하는 알고리즘
- 가중치가 있는 그래프에서 사용하며, 모든 간선의 가중치는 0 이상이어야 함
- **“지금까지 발견한 최단 거리 중 가장 짧은 정점을 확정해 나간다”**

## 사용 상황
- 단일 시작점 최단 경로
- 간선 가중치가 음수가 없는 그래프
- 예시 
  - 지도 최단 경로
  - 네트워크 지연 시간 
  - 게임 맵 이동 비용 계산

- ❌ 사용 불가 
  - 음수 가중치 존재 &rarr; 벨만-포드 사용

## 구현 포인트
#### 1. 거리 배열(dist[])
- 시작점: 0
- 나머지: INF로 초기화 

#### 2. 우선순위 큐 (PriorityQueue)
- 현재까지의 최단 거리가 가장 작은 정점을 먼저 꺼냄
- (거리, 정점) 형태로 저장

#### 3. 완화(Relaxation)
```java
if (dist[next] > dist[cur] + cost) {
	dist[next] = dist[cur] + cost;
}
```

#### 4. 중복 방문 방지
- if (현재 거리 > dist[정점]) continue;
- 방문 배열 없이도 처리 가능

## 시간복잡도
- 배열 기반: **O(V²)**
- 우선순위 큐 (Heap): **O((V + E) log V)**

## 실수 포인트
- 음수 가중치 사용
- visited 처리 잘못해서 더 짧은 경로를 막아버림
- PQ에서 꺼낸 값이 최신 거리인지 확인 안 함
- INF 값이 너무 작아 오버플로 발생
