# 크루스칼 (Kruskal) vs 프림 (Prim)

| 구분 | 크루스칼 (Kruskal) | 프림 (Prim) |
|----|----|----|
| 목적 | 최소 신장 트리(MST) 구성 | 최소 신장 트리(MST) 구성 |
| 접근 방식 | **간선 중심** | **정점 중심** |
| 선택 기준 | 가중치가 가장 작은 간선 | 현재 트리에서 나갈 수 있는 최소 간선 |
| 시작 정점 | 필요 없음 | 필요 있음 (아무 정점이나 가능) |
| 트리 확장 | 간선 하나씩 추가 | 정점 하나씩 확장 |
| 사이클 처리 | **유니온–파인드**로 판별 | `visited` 배열로 구조상 방지 |
| 핵심 자료구조 | 간선 정렬 + Union-Find | PriorityQueue |
| 시간복잡도 | **O(E log E)** | **O(E log V)** |
| 유리한 경우 | 간선 수가 적은 그래프 | 정점 수가 많은 그래프 |
| 구현 난이도 | 보통 | 쉬움 |

## 핵심 차이 한 줄 요약
- **크루스칼**: “싼 간선부터 고르고, 사이클만 피하자”
- **프림**: “현재 트리에서 가장 싸게 확장하자”

## 선택 기준 정리
- **간선 위주로 주어지고 간선 수가 적다** &rarr; 크루스칼
- **정점 수가 많고 인접 리스트 그래프다** &rarr; 프림
